## ✅ Fully-dynamic A4 editor (PUSHED to main `c1e8d7e`, 2026-06-28)

> User ask: *"in the reports, we need the a4 editor to be fully dynamic, so we can edit text, ensure correct page structure/breaks, and have much more control over the final pdf."* Decisions (AskUserQuestion): **edit directly on the page** (inline WYSIWYG; once hand-edited the user's text wins, "Reset to template" restores the data-bound render) + **auto-flow + insert breaks** (content flows into real A4 sheets; a manual page break forces one; cards/tables never split; export is a crisp multi-page PDF matching the screen).

- **`components/reports/ReportEditor.tsx` (NEW)** — the read-only preview iframe became an **editable, paginated A4 document**. Geometry A4@96dpi 794×1123, 60px (~16mm) margins, 28px on-screen gutter. The token-rendered report is split (head + `.sheet` body) and re-hosted in a `contenteditable` `#flow`; **spacer-based pagination** measures each top-level block and inserts invisible spacer divs to push overflow to the next sheet — **editable nodes are never moved, so the caret survives typing**. White `.pg-sheet` rectangles render behind the flow; **fit-to-width** via CSS `transform: scale()` (doesn't perturb `offsetTop`, so measurement stays exact). Toolbar: bold/italic/list (execCommand), **Insert page break** (`<hr class="pg-break">`), undo. Exports: **multi-page PDF** (html2canvas@scale2 of `#doc`, sliced per A4 sheet → jsPDF, unscaled to full res), browser **print**, standalone **HTML**. `ReportState.editedHtml` (new optional field on the existing `inputs` jsonb — **no schema change**) stores the override; an amber banner + **Reset to template** in `ReportStudio`.
- **Adversarial review (workflow, 2 dims × find→verify, 8 raised / 6 confirmed) — all 6 fixed + browser-verified:**
  1. **Stored XSS (high)** — the contenteditable body (paste/typed) was persisted to `editedHtml`, re-injected via iframe `srcdoc`, AND exported as a client-facing `.html`, all unsanitised. Fix: **`reports/sanitize.ts`** (DOMPurify, now a **direct dep**) sanitises at the single `captureBody` chokepoint + the iframe gains **`sandbox="allow-same-origin"`** (no `allow-scripts` → injected markup can't execute in the editing surface). Verified: a `<script>`/`<img onerror>`/`javascript:` payload is stripped from the persisted state AND the exported HTML, while the inline SVG chart + inline styles survive; sandbox blocked execution (no trip).
  2. **HTML-export geometry divergence (high)** — `getDocHtml()` re-wrapped the body in the template's `.sheet` (840px) while the editor paginates at 674px → downloaded HTML wrapped/broke differently than the screen. Fix: export now reproduces the editor's `#flow` geometry (794px / 60px → 674px content). Verified.
  3. **Oversized block (>1 page) (med)** — a single block taller than the printable area produced a negative/skipped spacer → content overlapped the sheet boundary and **dropped off the PDF**. Fix: `paginate()` advances `page` past every full page-height the block spans. Verified (2300px block → 5 sheets cover it, zero negative spacers).
  4. **Lost-edit-on-tab-switch (med)** — switching document tabs unmounts the studio; both debounce timers (editor 280ms, autosave 800ms) were cancelled, dropping the last edit. Fix: editor flushes the freshest body on a **layout-effect** unmount (only if the page was actually edited) → studio `persistNow` writes it into the in-memory workspace session synchronously (remount restores it) + a durable background save; a separate studio unmount cleanup flushes field-only edits. Verified: an edit made & switched away **inside** the debounce window survived.
  5. **`html2canvas` undeclared (med)** — only resolvable as jspdf's optional transitive dep. Fix: added `html2canvas` (+ `dompurify`) to `web/package.json` dependencies + lockfile.
  6. **Spacers editable (low)** — pagination spacers now `contenteditable="false"` + `aria-hidden`, matching `.pg-break`.
