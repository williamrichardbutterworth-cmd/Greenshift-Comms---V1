## 1. What Comms is now (product state, all shipped)

Five tabs:

- **Dashboard** — hero strip (as-of + derived one-line market summary, "N of M metrics live" chip, refresh button), an interactive **Price-history explorer** (`PriceExplorer.tsx`: Brent/Gas/Power × 3m/6m/12m via `/api/market/history`, with Latest/Change/High/Low/Average window stats), live metrics grouped **Gas / Power / Oil & storage** (cards with big formatted values, change-chips, sparklines + live-source dots), sorted generation-mix donut with centre label + share bars, attribution + indicative-pricing footer.

- **News** — tabbed hub: **Live feed · Headlines · Library**, in a two-column card grid with search, clamped summaries and clickable topic tags. Topic filter chips with live counts (geopolitics/policy/renewables/gas/power/oil/macro — deterministic keyword classifier, no AI cost). **Manage feeds** panel (toggle switches + enabled count; add/remove RSS sources; DB-backed, seeded from `providers/news/feeds.ts`). **Save** to library / **Pin** to persistent Headlines; live-feed refresh button. A prominent **"Add an article"** bar: paste any URL → page scraped (OG/meta) → **AI-summarised for an energy agent** → topic-classified → saved to Library (graceful fallback to meta description with no key).
- **Report** — the flagship. Opens on a **management overview** (`ReportHome.tsx`): searchable project grid (rename/delete on hover) + client-profile management rows (rename/delete + one-click **Start report** from any saved client). Opening a project enters the **Word-style A4 studio**: page-overview thumbnails (left, click-to-scroll, live-synced) · the A4 page (centre, clear "Page N" break separators at 1122px intervals) · collapsible setup panel (right: **Client profile / References & media / Your input**) · top bar (back-to-overview, Open-report ▾, rename, Versions, Save version, **Assemble**, Export ▾). Flow: **New report → client-profile step** (pick saved profile or create; optionally paste a call transcript that's mined to pre-fill fields + notes) → enter the project. References & media unifies saved articles (incl. add-by-URL) + uploaded files, with a filter box and counts; each item is tickable as draft context, insertable (＋), and **draggable onto the page — inserting at the drop point** (`posAtCoords`); desktop files drop straight in (images embed, PDFs/Word get text-extracted as context). The **context tray (snapshot/brief toggles, ticked references, extra notes) persists into `report_projects.context`** and restores on open. **Assemble** (with a staged progress overlay) sends weighted context buckets (client profile / market / evidence / your input) → the model returns an ordered *section list* (never raw doc JSON; embeds are named slots so it can't fabricate figures) → client builds the TipTap doc. Inline **Rewrite** menu (concise/expand/add a market figure/reword/regenerate) on any selection. Block inserts: market metrics table, price chart (brent/gas/power × 3m/6m/12m), custom data chart (with auto-caption "Analyse"), news evidence. Projects autosave (debounced) with a 20-deep version history; export is **client-side** branded PDF (jsPDF: tinted client-details cover panel, running header + "Page X of Y" footer) + Word (docx: title-page header/footer with live page numbers).
- **Admin Ideas** — submit/vote/status board + theme digest ("Summarise themes").

Compliance is wired throughout and must stay: indicative-pricing + not-financial-advice disclaimers, human-review-before-export note, source attribution (Elexon BMRS, National Gas, Yahoo) in dashboard + report footers.

## 3. Architecture

**Production (Vercel):** `comms/web` → Vite static build. `comms/api/[...path].ts` → ONE Node serverless function that dynamic-imports the **esbuild-bundled** Fastify app (`comms/server/dist/app.js`, built by `server`'s `npm run build`) and dispatches each request via `app.inject()` (Vercel pre-parses JSON bodies; we pass `req.body` as the inject payload). `comms/vercel.json` rewrites `/api/(.*) → /api/[...path]` (required for multi-segment paths) and `/((?!api/).*) → /index.html` (SPA fallback excluding `/api`).

**Local dev:** `server/src/index.ts` runs the same `buildApp()` long-lived on :8080 + a node-cron scheduler (`0 7 * * 1-5` pre-warms the daily brief — **local only**; serverless has no scheduler, hence the §9B cron item). `web` runs Vite (proxying `/api` → :8080). `.claude/launch.json` defines preview servers: **web :5273**, **api :8080**.

**Persistence pattern (used by all 5 stores):** `lib/supabase.ts#getSupabase()` returns a client when `SUPABASE_URL`+`SUPABASE_SERVICE_ROLE_KEY` are set, else `null` → store falls back to a local JSON file in `server/data/` (atomic `.tmp`+rename writes, in-memory cache). Same public interface either way. File bytes: Supabase Storage bucket `report-files` in prod, `server/data/uploads/` locally.

**AI pattern:** everything goes through `providers/ai` (`AIProvider` interface, `getAI()`, `generateText`/`generateJSON` + defensive `extractJSON`). All prompts live in **`server/src/services/prompts.ts`** (content ≠ code; the product "voice" + accuracy rules are there). User-facing AI endpoints (**assemble, edit, transcript-extract, article summarise, ideas digest, daily brief**) all **degrade gracefully** — no key/billing failure ever 500s; they return placeholders/skeletons/unchanged text.

**Report document model:** a single TipTap v2 doc (ProseMirror JSON, stored opaque in `report_projects.doc`). Prose = native nodes; structured content = four **atom** block nodes (`metricsTable`, `priceChart`, `customChart`, `newsList`) holding data in attrs, rendered by React node-views, plus `image`. AI assembly returns `{sections:[{kind:'text'|'embed',…}]}` and `web/src/lib/buildDocFromSections.ts` materialises the doc client-side. Export (`web/src/lib/exportReport.ts` + `serializeDoc.ts`) walks the doc → branded PDF/Word entirely in the browser (no Puppeteer); charts are stored as **data only** and re-rendered via `chartSvg.ts` → PNG at export time.

## 7. Hard-won gotchas (do NOT re-learn these)

1. **No Puppeteer on Vercel** — PDF/Word are client-side (`exportReport.ts`). jsPDF images need compression (`'FAST'`/`'MEDIUM'`).
2. **Bundle the backend** — esbuild `--bundle --format=esm --packages=external` → `server/dist/app.js`; the function imports that (plain TS imports don't resolve in the serverless ESM runtime).
3. **Route nested `/api/*`** — the explicit `"/api/(.*)"` rewrite is required; the SPA rewrite must exclude `/api`.
4. **Opus 4.7+ reject `temperature`/`top_p`/`top_k`** (sending them returns a 400); **Fable 5 additionally rejects an explicit `thinking:{type:'disabled'}`**. `claude.ts` omits all of them — safe across every supported model, so switching `CLAUDE_MODEL` between Opus and Fable needs no code change. Don't add them back.
5. **TipTap is pinned to v2 (`^2.27.2`)** — installing an extension without a version pulls v3 and breaks peer deps (happened with `extension-image`; install `@^2.27.2`). Import ProseMirror only via `@tiptap/pm/*` (duplicate-instance crash otherwise).
6. **`pdf-parse` is broken under ESM** (its debug self-test crashes on import; its exports map blocks the subpath workaround) — we use **`unpdf`**. Keep it.
7. **jsPDF hangs (synchronously, uninterruptibly) on degenerate images** — the exporter skips images < 8px. Don't remove that guard.
8. **`headlines` has no `note` column** — `addHeadline` must insert columns explicitly, never spread the shared article fields.
9. **Uploads are base64 JSON** (simple through `inject()`); Fastify `bodyLimit` is 8MB but **Vercel Hobby caps request bodies ~4.5MB** → practical upload ≈ 3MB despite the store's 6MB cap.
10. **Charts are stored as data, never PNG** — node attrs hold `{t,v}`/`{label,value}` points; `chartSvg.ts` renders at export. Keep this invariant (tiny docs, small Supabase rows).
11. **A4 constants must stay in sync** (794×1122px @96dpi): the **1122 height** appears in `index.css` (`.report-sheet`), `CommsEditor.tsx` (`A4_PAGE_H`) and `PageOverview.tsx`; the **794 width** in `index.css` and `PageOverview.tsx`. Change one → change all.
12. **Dev-server crash recovery**: `tsx watch`'s child can die on an import crash while the watcher parent lives — the preview tool then thinks it's "running". `preview_stop` + `preview_start` to actually restart.
13. **Supabase + file fallback** — `getSupabase()` returns null without env; every store must keep its dual path so local dev needs no Supabase.
14. **`web/tsconfig.node.json`** must not be `composite`+`noEmit` together. Also: don't use `import.meta.env` in web code without vite/client types — it broke `tsc -b` once.
15. **Git/auth**: the keychain credential is the wrong account; one-off PAT pattern in §2. Never store tokens in the repo (it's public).
16. **No "AI" in UI copy** (user requirement). The model/provider name is never surfaced to users either.
17. **Serverless time budget:** the API function is pinned to `maxDuration: 30` (seconds) / 1024MB in `vercel.json`. Brief generation already uses ~20s; assembles run close to the cap. Timeouts manifest **only in prod**, never in local dev. Raising beyond 60s needs Vercel Pro.
18. **Recharts entry animations must stay disabled** (`isAnimationActive={false}` on every series — MetricCard, PriceExplorer, the Dashboard pie, and the PriceChart/CustomChart editor nodes). With animations on, charts freeze mid-draw in some environments and react-smooth floods the console with flushSync warnings. Set it on any new chart too.
